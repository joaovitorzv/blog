4 Conceitos básicos de POO

07 de janeiro de 2022

Neste post utilizarei os exemplos em python mas os conceitos de POO (Programação Orientada a Objetos) são "universais".

1 - Encapsulamento (encapsulation):

O objetivo de "encapsular" o código é fazer com que, tudo o que, esteja dentro de determinado bloco de código só possa ser manipulado dentro de si, em python para definir esse bloco de código utilizamos a palavra classe. para exemplificar melhor vamos criar uma classe pessoa 

⁠class People {
⁠    ⁠def __init__(self):
⁠}

todo ser humano é da espécie Homo Sapiens e tem diversos atributos (nome, idade, peso etc) e provavelmente também realiza ações como andar e falar, essas ações em POO podem ser chamadas de métodos

⁠class People:
⁠⁠    species = 'Homo Sapiens' 
⁠
    def __init__(self, name):
⁠⁠    ⁠    self.name = name
⁠
⁠    def speak(self, msg):
⁠⁠    ⁠    print("{who} disse: {msg}".format(who=self.name, msg=msg))
⁠
    def walk(self):
⁠⁠    ⁠    print(People.species + " esta andando...")

como você pode ver a cima, criamos uma classe People com o atributo species como 'Homo Sapiens' , definimos um atributo nome que vai ter seu valor atribuído no momento em que instanciarmos algum objeto People e também criamos os métodos speak e walk, aqui já podemos ver o encapsulamento em ação, todos os dados que eu manipular dentro da classe deverão relacionados a uma "pessoa"

⁠if __name__ == '__main__':
⁠⁠    person = People(name='joao')
⁠⁠    print(person.name) # "joao"
⁠⁠    person.speak('as fumaças falarão por mim') # "joao disse: as fumaças falarão por mim"
⁠⁠    person.walk() # "Homo Sapiens esta andando..."

note que na instancia person temos acesso aos métodos e também ao atributo name da pessoa. Agora imagine que existe um método que possibilita a pessoa de comprar algo, para o bem da simplicidade vamos definir, que, para realizar a compra ela vai precisar passar o número do seu CPF e o nome de algum item

⁠class People:
    [...]

    def __init__(self, name, cpf):
⁠⁠    ⁠    self.name = name
⁠⁠    ⁠    self.cpf = cpf

    [...]

    def buy(self, item):
⁠⁠    ⁠    if (self.cpf and item):
⁠⁠    ⁠        print("{who} comprou {item}".format(who=self.name, item=item))
⁠
if __name__ == '__main__':
⁠⁠    person = People(name='joao', cpf='1234')
⁠⁠    person.buy('agua') # joao comprou agua
⁠⁠    print(person.cpf) # 1234

como você pode ver acima, é possível por meio da instância person acessar o CPF. Aqui o encapsulamento entra em ação mais uma vez, como o CPF é um dado sigiloso não queremos que ele fique disponível para ser acessado por ai, então vamos fazer com que esse atributo seja privado

⁠  [...]

def __init__(self, name, cpf):
⁠    self.name = name 
⁠    self.__cpf = cpf

  [...]

def buy(self, item):
⁠    if (self.__cpf):
⁠        print("{who} comprou {item}".format(who=self.name, item=item))

if __name__ == '__main__': 
⁠    person = People(name='joao', cpf='1234')
⁠    person.buy('agua') # joao comprou agua⁠
⁠⁠    print(person.__cpf) # AttributeError: 'People' object has no attribute '__cpf'
⁠⁠    print(person.cpf) # AttributeError: 'People' object has no attribute 'cpf'

dessa maneira os dados que fazem sentido serem manipulados apenas dentro da instancia ficam "blindados" dentro da classe.

Também poderíamos criar métodos privados caso necessário.

2 - Abstração

Como o código é encapsulado temos uma melhor abstração da codebase no geral, por exemplo: normalmente quando estamos andando ou falando nosso cérebro não está pensando quantos centímetros vamos abrir a perna ou a velocidade que falamos, apenas executamos essa ação de uma forma "automática", aqui algo parecido acontece, quando chamamos o método person.speak('blah blah blah') da instancia person toda ação simplesmente é executada

3 - Herança (Inheritance)

Esse conceito nos permite com que o código possa ser reutilizado de uma forma inteligente dentro da nossa codebase

⁠# em outro arquivo importamos 
⁠# a classe que desejamos herdar
⁠from people import People
⁠
⁠class Superhero(People):
⁠    species = 'superhero'
⁠
    def __init__(self, name):
⁠        self.name = name
⁠
    def invisibility(self):
⁠        print('estou invisível')
⁠
if __name__ == '__main__':
    ⁠hero = Superhero('ben 10')
⁠    print(hero.species) # superhero
⁠    hero.speak('estou falando') # ben 10 disse: estou falando
⁠    hero.invisibility() # estou invisível

acima criamos uma nova classe Superhero e herdamos a classe People, dessa forma é possivel reutilizar o método speak criado na classe pai (People)

4 - Polimorfismo (Polymorphism)

um exemplo bem pratico de polimorfismo é o operador '+', é possivel utiliza-lo para realizar operações "distintas"

x = 1
y = 2

sum = x + y # 3
# ou
str = x + ' ' + y # "1 3"

mas como estamos falando de POO, vamos utilizar polimorfismo em nossas classes

class Superhero():
    species = 'superhero'
    def __init__(self, name):
        self.name = name

    def invisibility(self):
        print('estou invisivel')

    def dna(self):
        return 'super-dna'

class Robot():
    species = 'iron'

    def __init__(self, name):
        self.name = name

    def dna(self):
        return "01101011"

if __name__ == '__main__':
    hero = Superhero('ben 10')
    robot = Robot('bot')

    for i in (hero, robot):
        print(i.dna()) # super-dna, 01101011

criei a classe robot para deixar o exemplo mais dinâmico, como você pode ver as classes não são conectadas de nenhuma forma e nem herdam uma classe em comum, porem, elas compartilham de um método com o mesmo nome dna, dessa forma (sabendo que eles existem) podemos realizar a operação acima, sem se preocupar. Isso só é possivel por causa do polimorfismo.

em alguns casos quando herdando algum método que não resolve completamente o seu problema é necessário re-implementar o método herdado, isso é chamado de method overriding

class Robot(People):
    species = 'iron'

    def __init__(self, name):
        self.name = name

    def dna(self):
        return "010101010101"

    def speak(self):
        print("não quero conversar.")


if __name__ == '__main__':
    robot = Robot('bot')

    robot.speak() # "não quero conversar."

no exemplo acima a classe Robot herda a classe People mas como o robot não está muito amigável o método speak foi re-implementado e se comporta de uma forma distinta da sua classe pai.
